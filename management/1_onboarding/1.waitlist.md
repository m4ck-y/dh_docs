# 1. Waitlist

Gestión de leads interesados. Los datos se almacenan en **MongoDB** dentro de `dh_onboarding_back`.

## Ciclo de vida del lead

```
ACTIVE → INVITED → CONVERTED
  │
  ├── BLOCKED
  └── EXPIRED
```

---

## 1.1 Registro de lead

**Actor**: Lead (desde landing page / formulario externo)

```
POST /v1/waitlist
Servicio: dh_onboarding_back
```

**Body:**
```json
{
  "client_name": "Juan Pérez",
  "email": "juan@example.com",
  "source": "landing_page"
}
```

**Efectos:**
- Se crea documento en MongoDB con `status: ACTIVE`
- VitalTrace recibe evento `lead_registered`
- PulseCore envía email de confirmación (`POST /v1/waitlist/confirmation`)

**Respuesta:** `201` con `ApiResponseSingle`

---

## 1.2 Verificar email registrado

**Actor**: Frontend (validación en tiempo real antes de enviar el formulario)

```
GET /v1/waitlist/check/{email}
Servicio: dh_onboarding_back
```

**Respuesta:**
```json
{
  "data": { "registered": true, "status": "ACTIVE" }
}
```

---

## 1.3 Listar leads

**Actor**: Admin

```
GET /v1/waitlist?status=ACTIVE&source=landing_page&page=1&limit=50
Servicio: dh_onboarding_back
```

**Respuesta:** `200` con `ApiResponsePaginated`

---

## 1.4 Invitar lead a iniciar onboarding

**Actor**: Admin

```
POST /v1/waitlist/{email}/invite
Servicio: dh_onboarding_back
```

**Efectos:**
- Genera `invite_token` seguro (`secrets.token_urlsafe(32)`)
- Guarda token en MongoDB con expiración de 7 días
- `status` → `INVITED`, `timestamps.invited_at` → ahora
- VitalTrace recibe evento `lead_invited`
- PulseCore envía email de invitación con el link (`POST /v1/waitlist/invite`)

**Respuesta:**
```json
{
  "data": {
    "email": "juan@example.com",
    "status": "INVITED",
    "invite_token": "tok_abc123...",
    "token_expires_at": "2026-05-03T10:30:00Z"
  }
}
```

---

## 1.5 Email de confirmación (PulseCore)

Disparado automáticamente en el paso 1.1. El frontend no llama esto directamente.

```
POST /v1/waitlist/confirmation
Servicio: app_message_sender (PulseCore)
```

**Body:**
```json
{
  "email": "juan@example.com",
  "user_name": "Juan Pérez"
}
```

---

## 1.6 Email de invitación (PulseCore)

Disparado automáticamente en el paso 1.4. El frontend no llama esto directamente.

```
POST /v1/waitlist/invite
Servicio: app_message_sender (PulseCore)
```

**Body:**
```json
{
  "email": "juan@example.com",
  "client_name": "Juan Pérez",
  "invite_token": "tok_abc123...",
  "token_expires_at": "2026-05-03T10:30:00Z"
}
```

El link en el email apunta a: `{WEBSITE_URL}/onboarding/start?token=tok_abc123...`
