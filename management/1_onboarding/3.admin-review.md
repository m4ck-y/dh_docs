# 3. Admin Review

El admin evalúa los applicants que completaron el onboarding (`status: SUBMITTED`).
Los datos viven en PostgreSQL bajo `api_core`.

## Flujo de revisión

```
SUBMITTED → UNDER_REVIEW → APPROVED → usuario activo
                         ↓
                       REJECTED → applicant notificado
```

---

## 3.1 Listar applicants pendientes de revisión

```
GET /api/v1/applicants?status=SUBMITTED
Servicio: api_core
```

Retorna lista de `people.person` con `verification_status: SUBMITTED`.

---

## 3.2 Ver detalle del applicant

```
GET /api/v1/applicants/{id_person}
Servicio: api_core
```

Incluye datos personales, documentos y estado de cada documento.

---

## 3.3 Aprobar documento individual

```
POST /api/v1/applicants/{id_person}/documents/{id_document}/approve
Servicio: api_core
```

**Efectos:**
- `expedient.document.verification_status` → `APPROVED`

---

## 3.4 Rechazar documento individual

```
POST /api/v1/applicants/{id_person}/documents/{id_document}/reject
Servicio: api_core
Body: { "reason": "Documento ilegible" }
```

**Efectos:**
- `expedient.document.verification_status` → `REJECTED`

---

## 3.5 Aprobar applicant (promoción a usuario)

```
POST /api/v1/applicants/{id_person}/approve
Servicio: api_core
```

**Efectos:**
- `people.person.verification_status` → `APPROVED`
- Se crea `iam.membership` con el rol asignado (PATIENT, DOCTOR, etc.)
- PulseCore envía email de bienvenida al nuevo usuario:

```
POST /v1/notifications/welcome
Servicio: app_message_sender (PulseCore)
Body: {
  "email": "juan@example.com",
  "user_name": "Juan Pérez",
  "login_url": "https://app.com/login"
}
```

---

## 3.6 Rechazar applicant

```
POST /api/v1/applicants/{id_person}/reject
Servicio: api_core
Body: { "reason": "Documentación incompleta" }
```

**Efectos:**
- `people.person.verification_status` → `REJECTED`
- Se notifica al applicant (pendiente de implementar)

---

## 3.7 Solicitar información adicional

```
POST /api/v1/applicants/{id_person}/request-info
Servicio: api_core
Body: { "message": "Por favor sube una foto más clara de tu identificación." }
```
