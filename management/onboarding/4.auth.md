# 4. Auth

Capa de autenticación. Solo opera sobre usuarios con `verification_status: APPROVED`.
Servicio responsable: `app_auth` (pendiente de extraer de `api_core` — ver TASK-004).

## Reglas críticas

- Solo usuarios con `verification_status: APPROVED` pueden hacer login.
- El access token tiene vida corta (15 min).
- El refresh token tiene vida larga (7-30 días).
- MFA (OTP) puede ser requerido en login o en acciones sensibles.

---

## 4.1 Login

```
POST /v1/auth/login
Servicio: app_auth
```

**Body:**
```json
{
  "email": "juan@example.com",
  "password": "MySecurePass123!"
}
```

**Respuesta:**
```json
{
  "data": {
    "access_token": "eyJ...",
    "refresh_token": "eyJ...",
    "token_type": "Bearer",
    "expires_in": 900
  }
}
```

**Si se requiere MFA:**
```json
{
  "data": {
    "mfa_required": true,
    "challenge_id": "uuid"
  }
}
```

---

## 4.2 Verificar MFA en login

Si el login requiere OTP, el applicant recibe un código por el canal configurado (EMAIL/SMS).

**PulseCore envía el código:**
```
POST /v1/otp
Servicio: app_message_sender (PulseCore)
Body: {
  "destination": "juan@example.com",
  "code": "A1B2C3",
  "purpose": "LOGIN",
  "expiry_minutes": 5,
  "channel": "email"
}
```

**Verificar:**
```
POST /v1/auth/mfa/verify
Servicio: app_auth
Body: { "challenge_id": "uuid", "code": "A1B2C3" }
```

---

## 4.3 Refresh de token

```
POST /v1/auth/refresh
Servicio: app_auth
Body: { "refresh_token": "eyJ..." }
```

---

## 4.4 Logout

```
POST /v1/auth/logout
Servicio: app_auth
Headers: Authorization: Bearer {access_token}
```

Revoca el refresh token activo.

---

## 4.5 Obtener usuario actual

```
GET /v1/auth/me
Servicio: app_auth
Headers: Authorization: Bearer {access_token}
```

---

## 4.6 Recuperación de contraseña

### Solicitar reset
```
POST /v1/auth/forgot-password
Body: { "email": "juan@example.com" }
```

**PulseCore envía código:**
```
POST /v1/otp
Body: {
  "destination": "juan@example.com",
  "code": "B2C3D4",
  "purpose": "RESET_PASSWORD",
  "expiry_minutes": 15,
  "channel": "email"
}
```

### Confirmar reset
```
POST /v1/auth/reset-password
Body: {
  "email": "juan@example.com",
  "code": "B2C3D4",
  "new_password": "NuevoPass456!"
}
```

## Códigos de error

| HTTP | Código interno | Significado |
|---|---|---|
| 401 | 10 | Sesión no iniciada |
| 401 | 11 | Token expirado |
| 401 | 12 | Token inválido |
| 401 | 13 | Usuario bloqueado |
| 401 | 14 | Refresh token inválido o revocado |
| 403 | 20 | Sin permisos para este recurso |
| 400 | 30 | Email inválido |
| 400 | 31 | Contraseña débil |
| 400 | 32 | OTP incorrecto |
