# TASK-002: Implementación de Microservicio Waitlist (MongoDB)

## Descripcion
Desarrollo del microservicio encargado de gestionar los leads interesados (Waitlist) utilizando MongoDB como base de datos documental. Este servicio es el punto de entrada inicial para el flujo de onboarding.

## Objetivos
- [ ] Definir el esquema de la colección `waitlist` en MongoDB.
- [ ] Implementar el CRUD básico de leads (Create, Read, List).
- [ ] Implementar la lógica de generación y almacenamiento de tokens de invitación.
- [ ] Exponer endpoints para que el administrador pueda invitar leads.
- [ ] Integrar el microservicio con el sistema de logging/tracing.
